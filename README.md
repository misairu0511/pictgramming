# 🎭 ピクトグラミング (Pictgramming)

ブラウザ上で動作するプログラミング学習ツールです。
ユーザーは Java風のテキスト構文を記述することで、キャンバス上に描かれたピクトグラム（人型のキャラクター）を直感的に動かすことができます。

## 🚀 起動方法

```bash
npm install
npm start
```

ブラウザで `http://localhost:3000` を開いてください。
（サーバー起動時に自動で現在時刻名のログフォルダが生成されます）

## 🧩 使い方と特徴

* **テキストプログラミング**: エディタに `移動(50);` などの命令や、`for (int i=0; i<3; i++)` などのJava標準の構文を入力して実行します。
* **アニメーションと操作**: 実行ボタンを押すとピクトグラムがアニメーションしながら動きます。途中で「一時停止 / 再開」や「停止（キャンセル）」が可能です。
* **アイテムの運搬とゴール判定**: ピクトグラムの手をうまく動かしてヒヨコの画像を「掴む」ことでアイテムを運べます。指定されたGOALエリアに運び入れると、終了時に判定結果（ゴールしたか、持ちっぱなしか等）が表示されます。
* **実行ログの保存**: プログラムを実行するたびに、動作したコマンドと結果が `logs/（起動時刻）/execution.log` に自動保存されます。

## 🔧 コマンド一覧

| 日本語コマンド | 旧名称・機能説明 |
|--------|------|
| `移動(数値);` | ピクトグラム全体を前後に移動。プラスで前進、マイナスで後退。 (旧: `move`) |
| `回転(数値);` | ピクトグラム全体を回転。プラスで右回転、マイナスで左回転。 (旧: `rotate`) |
| `部位回転("部位名", 数値);` | 指定した関節だけを回転。 (旧: `rotatePart`)<br>※部位名は `"左腕"`, `"右足"`, `"左膝"` など日本語または英語。 |
| `掴む();` | 手の先の当たり判定内にヒヨコがある場合、画像を掴む。 (旧: `grab`) |
| `離す();` | 掴んでいる画像をその場に離す。 (旧: `release`) |

## 📦 ファイル構成

```
pictgramming/
├── server.js          # Node.js サーバー (API、ログ保存処理)
├── package.json
├── logs/              # サーバー起動ごとに作成される実行ログ保存ディレクトリ
├── public/
│   ├── index.html     # メインUI（エディタ、キャンバス、操作ボタン）
│   ├── css/
│   │   └── style.css  # スタイル
│   └── js/
│       ├── engine.js  # ピクトグラムの描画、アニメーション、ゴール判定エンジン
│       └── app.js     # Java構文のトランスパイル、UIイベント制御、ログ送信
└── README.md
```
