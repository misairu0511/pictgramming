# ピクトグラミング (Pictgramming) プロジェクト仕様書

このドキュメントは、本プロジェクトに新しく参加する他のAIエージェントや開発者が、プロジェクトの全体像やコードの構造、主要な機能について迅速に理解できるように作成されたものです。

## 1. プロジェクト概要
「ピクトグラミング」は、ブラウザ上で動作するプログラミング学習ツールです。ユーザーは Java 風の短い構文を記述することで、キャンバス上に描かれたピクトグラム（人型のキャラクター）を動かすことができます。

## 2. ディレクトリ・ファイル構成
```
pictgramming/
├── index.html       # メインのUI画面。エディタ、キャンバス、ヘルプテキストを含む
├── public/
│   ├── css/
│   │   └── style.css  # 画面のスタイリング
│   └── js/
│       ├── app.js     # UIの制御およびJava構文のパース（解析）ロジック
│       └── engine.js  # ピクトグラムの描画およびアニメーション実行エンジン
└── server.js        # （存在する場合）ローカルサーバー起動用
```

## 3. 主要なコマンド一覧 (2026/06 最新仕様)
以前は `moveForward` や `rotateRight` のように方向ごとにコマンドが分かれていましたが、コードを簡略化するため、正負の数値を用いて1つのコマンドに統合されています。

* **`移動(数値)`**
  * ピクトグラム全体を指定された距離だけ前後に移動させます。
  * `正の数` で前進、`負の数` で後退します。
* **`回転(数値)`**
  * ピクトグラム全体を回転させます。
  * `正の数` で右回転、`負の数` で左回転します。
* **`部位回転("体の部位", 数値)`**
  * 指定した体の部位のみを回転させます。
  * `正の数` で右回転、`負の数` で左回転します。
  * 指定可能な部位（文字列）: `"head"`, `"body"`, `"leftArm"`, `"rightArm"`, `"leftElbow"`, `"rightElbow"`, `"leftLeg"`, `"rightLeg"`, `"leftKnee"`, `"rightKnee"` （※日本語での指定も可能: `"左腕"`, `"右腕"`, `"左肘"` など）
* **`掴む()`**
  * 手の先の当たり判定（半径35pxの円形）内に画像（ヒヨコなど）がある場合、その画像を掴みます。掴んだ後は手の動きに合わせて画像も移動・回転します。
* **`離す()`**
  * 掴んでいる画像をその場に離します。

## 4. 各モジュールの役割

### `app.js` (フロントエンドコントローラー & パーサー)
* **イベントリスナー**: 実行ボタン、リセットボタンのクリックイベントや、Canvas上のマウスホバーイベント（部位名表示）を管理します。
* **トランスパイル処理 (`transpileToJava`)**: 
  * 入力されたテキスト（Java風の構文）から、`移動()` や `for` などの命令を抽出・置換します。
  * 非同期実行（アニメーション）のために `await picto.移動()` のように書き換え、裏側でJavaScriptとして実行させます。

### `engine.js` (`PictoEngine` クラス)
* **描画 (`draw`, `drawPicto` など)**: HTML5 Canvas 2D API を使用して、グリッド、軌跡、そしてピクトグラム自身を描画します。部位ごとの回転角度は `this.state.parts` で管理されます。
* **実行 (`execute`, `run`)**:
  * `app.js` から渡されたコマンドオブジェクトの配列を非同期的に順次実行します。
  * `move` なら `animateMove()`、`rotate` なら `animateTurn()`、`rotatePart` なら `animatePartRotate()` を呼び出し、内部の数値を変更しながら `requestAnimationFrame` でアニメーションを描画します。
  * アニメーションの補間には `lerp` (線形補間) を使用しています。

## 5. 今後の開発・改修時の注意点
* コマンドを追加・変更する場合は、**`index.html` (ヘルプテキスト)**、**`app.js` (パーサーの許可リストと引数チェック)**、**`engine.js` (実行関数の分岐とアニメーション処理)** の3箇所をセットで変更する必要があります。
* ピクトグラムの部位の当たり判定（マウスホバー時のツールチップ表示用）は `engine.js` の `getPartAt` メソッド内に座標ベースでハードコーディングされています。部位のサイズや描画位置を変更する場合は、この当たり判定も合わせて修正してください。
